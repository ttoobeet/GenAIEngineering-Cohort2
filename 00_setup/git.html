üì• Getting Started
git clone https://github.com/user/repo.git
Downloads entire repository from GitHub to your local machine. Creates a new folder with project name.
git init
Initializes a new Git repository in current folder. Use this to start tracking an existing project.
git remote add origin URL
Links your local repository to a GitHub repository. 'origin' is the conventional name for main remote.
üîÑ Syncing Changes
git pull origin main
Downloads and merges changes from GitHub to your local repository. Combines fetch + merge in one command.
git fetch origin
Downloads changes from GitHub without merging them. Lets you see what's new before integrating.
git push origin main
Uploads local commits to GitHub repository. Makes your changes available to others.
üåø Branching & Merging
git branch feature-name
Creates a new branch from current branch. Use for developing new features in isolation.
git checkout feature-name
Switches to specified branch. Changes your working directory to that branch's state.
git checkout -b feature-name
Creates a new branch AND switches to it in one command. Shortcut for branch + checkout.
git merge feature-name
Merges specified branch into current branch. Combines changes from both branches.
git branch -d feature-name
Deletes branch after it's been merged. Use -D to force delete unmerged branches.
üìù Basic Workflow
git add .
Stages all modified files in working directory to staging area. Use "git add filename" for specific files.
git commit -m "message"
Creates a snapshot of staged changes with a descriptive message. This saves your changes to local repository.
git push origin main
Uploads local commits to GitHub repository. Makes your changes available to others.
‚¨áÔ∏è Getting Updates
git pull --rebase origin main
Downloads changes and replays your commits on top. Keeps cleaner history than merge commits.
git pull upstream main
Gets updates from original repository when you've forked a project. Keeps your fork synchronized.
üîÑ Undoing Changes
git reset HEAD
Unstages files from staging area back to working directory. Files remain modified but not staged.
git reset --soft HEAD~1
Undoes last commit but keeps changes in staging area. Good for fixing commit messages.
git reset --hard HEAD~1
‚ö†Ô∏è DANGER: Completely removes last commit and all changes. Cannot be undone!
üîç Viewing Changes
git diff
Shows line-by-line differences between working directory and last commit. See what you've changed.
git diff --staged
Shows differences between staging area and last commit. See what will be committed.
git status
Shows current state: which files are modified, staged, or untracked. Your Git health check.
üõ°Ô∏è File Management
Create .gitignore
Creates file to specify which files Git should ignore. Essential for excluding sensitive data and build files.
git rm filename
Removes file from both working directory and Git tracking. Use --cached to only remove from Git.
üìä Branch Info & History
git branch
Lists all local branches. Current branch is marked with *. Add -r for remote branches.
git branch -a
Shows all branches (local and remote). Useful to see what branches exist on GitHub.
git log --graph --oneline
Visual commit history showing branch merges and divergences in a tree format.
git show-branch
Shows branches and their commits in a detailed tree view. Great for understanding branch relationships.

üì• Getting Started with Git Clone

Starting with an existing repository:

# 1. Clone repository from GitHub
git clone https://github.com/username/repository.git
cd repository

# 2. Make changes and track them
git add .
git commit -m "Your first contribution"

# 3. Push changes back to GitHub
git push origin main

# 4. Stay updated with latest changes
git pull origin main

üîê Authentication: When prompted for password, use your Personal Access Token (PAT) instead of GitHub password.

üåø Branching Workflow Example

Feature Branch Workflow:

# 1. Create and switch to feature branch
git checkout -b feature/user-login

# 2. Make changes and commit
git add .
git commit -m "Add user login functionality"

# 3. Push feature branch to GitHub
git push origin feature/user-login

# 4. Switch back to main and merge
git checkout main
git merge feature/user-login

# 5. Delete feature branch (cleanup)
git branch -d feature/user-login
git push origin --delete feature/user-login
